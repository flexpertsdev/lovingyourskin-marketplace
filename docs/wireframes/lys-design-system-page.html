Loving Your Skin - Design System

Design Principles

1. Mobile-First

Design for mobile screens first, then enhance for larger viewports. Touch-friendly interfaces with minimum 48px tap targets.

2. Minimalist Elegance

Clean layouts with generous whitespace. Korean beauty aesthetic with subtle, sophisticated elements.

3. Clear Hierarchy

Information architecture that guides users naturally. Progressive disclosure for complex workflows.

4. Accessibility

WCAG AA compliant. High contrast ratios, keyboard navigation, and screen reader support.

Responsive Behavior

Breakpoint System

/* Mobile First Approach */
.component {
  /* Mobile styles (default) */
  padding: 16px;
  font-size: 14px;
}

@media (min-width: 768px) {
  .component {
    /* Tablet styles */
    padding: 20px;
    font-size: 16px;
  }
}

@media (min-width: 1024px) {
  .component {
    /* Desktop styles */
    padding: 24px;
    font-size: 16px;
  }
}

/* Utility Classes */
.mobile-only { display: block; }
.tablet-up { display: none; }
.desktop-only { display: none; }

@media (min-width: 768px) {
  .mobile-only { display: none; }
  .tablet-up { display: block; }
}

@media (min-width: 1024px) {
  .desktop-only { display: block; }
}

Implementation Notes for Claude Code

Technical Architecture

CSS Architecture

  • Use CSS custom properties for theming
  • Implement utility-first approach with Tailwind CSS
  • Component-scoped styles with CSS Modules
  • PostCSS for vendor prefixing

Component Structure

  • Atomic design methodology
  • Compound components for complex UI
  • Prop-based variants
  • Composition over inheritance

State Management

  • Local state for component UI
  • Global state for user/cart/auth
  • Server state with React Query
  • URL state for filters/pagination

Performance

  • Code splitting by route
  • Lazy load images and components
  • Virtual scrolling for long lists
  • Service worker for offline support
